# RFB Android v7.5 交付说明

## 直接使用

- 新安装或从 v7.0～v7.4 覆盖升级：安装 `dist/RoguelikeFansBand-Android-v1.3.0.6-v7.5-universal-debug.apk`，不要先卸载旧版。
- 已安装早期 v6 测试包：运行根目录 `升级安装v7并保留存档.bat`。
- 自己跟进上游新版：Windows 双击 `打开图形化构建器.bat`，保持源码策略 `latest` 后构建。

## v7.5 存档目录与诊断

- “存档槽”打开可滚动目录，每个槽位直接提供“打开”和“新游戏”；当前运行槽提供“继续”。
- “新建存档并开始新游戏”会先要求一个安全槽名，然后以原版 `-n -u槽名` 模式启动。
- 切换槽位仍先通过原版 Ctrl+X 保存当前角色，再在干净 Android 进程中载入目标槽。
- 正常退出不显示诊断目录；只有异常退出或启动错误才显示日志路径。
- 启动日志最多保留当前与上一份各 256KB，单条异常也会截断；健康退出后两份都会自动清理。

## 操作区与 HUD

- 游戏终端仍为操作区保留真实宽度，最右侧游戏信息不会被栏位覆盖。
- 操作区只是统一 HUD 的背景和留白，不单独拥有按钮；任意按钮都可放在游戏区或操作区。
- 进入布局模式后，拖按键移动，轻点按键编辑；拖操作区主体换到左/右侧，拖黄色分界调节宽度。
- 操作区可修改背景显示、宽度、颜色、形状和透明度；按钮可分别修改文字、动作、宽高、形状、颜色、透明度与连发。
- 编辑按钮时可直接输入 X/Y 坐标（左上角为 0,0，单位 dp）；相同 X 或 Y 可快速对齐，保存时按按钮当前尺寸校验范围。
- 布局模式顶部会临时出现“完成、按键、按键管理、操作区”工具条，退出后完全隐藏。
- Android 返回键始终可以打开游戏菜单，避免删光按钮后失去恢复入口。
- v7.2 的原固定九宫格和界面控制会自动迁移成可编辑按钮，旧栏宽也会迁移且只执行一次。

## 输入、键盘与存档

- 九方向采用原生 `789456123`，方向键和自定义移动键可长按连发，所有触控键有视觉、缩放、按键音和触感反馈。
- 大键盘直接发送每次按键，含 Ctrl/Shift 下一键与锁定；它只占游戏区。打开时游戏区 HUD 键暂时隐藏并禁用，操作区 HUD 键继续可用，关闭后自动恢复。
- “游戏菜单 → 存档槽”可创建、打开或重新开局。切换时先通过原版 Ctrl+X 保存，再在干净进程载入目标槽。

## 最新版构建策略

- 默认解析上游 `/releases/latest`，只接受正式、非草稿、非预发布 Release。
- 支持固定标签、本地源码与离线复用最后一次成功缓存。
- 每次重新生成 Android 版本、资源指纹和兼容性记录；关键上游结构变化会明确中止，不静默输出坏包。
- 首次使用构建器需要联网下载便携 JDK、Python、Android SDK/NDK、CMake 与 Gradle；以后复用工程目录内缓存。

## 发布提示

工程内固定 Debug 密钥用于连续测试和覆盖升级。如果公开发布，请构建未签名 Release APK，用自己的正式密钥签名并永久保管；后续公开升级必须继续使用同一密钥。
