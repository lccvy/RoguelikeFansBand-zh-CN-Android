## 已确认的构建结果

1. 构建器默认策略仍为 `latest`，本次缓存解析到上游正式版 `v1.3.0.6`，并未把该游戏版本写死到工具中。
2. 动态版本为 `versionCode=103000066`、`versionName=1.3.0.6-android-v7.5`，高于 v7.4 的 `103000065`。
3. 全部 Java 源码使用 Java 17 与 Android 36 `android.jar` 无警告编译；D8 生成 141,484 字节的 `classes.dex`。
4. `tools/check_project.py` 和全部 `tools/tests/test_*.py` 通过；当前容器没有 clang，因此两个可选 C 编译烟雾测试按设计跳过，其余补丁回归断言通过。
5. 游戏终端与操作区仍物理分栏，操作区宽度不会覆盖终端最右侧信息；HUD 覆盖整块窗口，按钮可跨分界自由摆放。
6. 操作区不再拥有固定按钮。其背景、宽度、颜色、形状、透明度和左右停靠位置可保存；布局模式支持拖主体换边、拖黄色分界调宽。
7. 原固定九宫格与界面控制在 v7.2 升级时一次性迁移为普通 HUD 键；旧操作区宽度也会迁移。用户之后可增删至 0～64 键。
8. 按键编辑器可输入以横屏左上角为原点的 X/Y dp 坐标；保存时按当前屏幕和按钮尺寸校验并换算回响应式归一化位置，无需迁移旧布局。
9. 正常游玩界面不再常驻书名、HUD 数量或“可拖拽”等说明；只有首次使用和进入布局模式时显示短暂提示，编辑工具条仅在布局模式出现。
10. 大键盘仍是游戏区内的非模态浮层。打开时只隐藏并禁用中心点位于游戏区的 HUD 键，操作区 HUD 键保持显示和可点击；关闭后自动恢复。
11. 方向键继续使用 `789456123`，支持长按连发、按下反馈和 Ctrl/Shift 直通输入。
12. Android 返回键被保留为安全菜单入口，即使用户删除全部 HUD 键也能重新进入编辑器。
13. 存档槽改为独立滚动目录，每个槽位有“打开”和“新游戏”，当前槽有“继续”；新建槽明确以新游戏模式进入。
14. 正常返回码 0 不再显示诊断目录，并清空当前/上一份日志；异常返回才显示路径。两份日志各限制 256KB，写入前轮换且超长单条会截断。
15. 存档切换仍通过原版 Ctrl+X 保存后在干净进程切换，未改动上游存档格式。
16. APK 已通过 ZIP 完整性、Manifest、资源元数据、三 ABI、APK v2/v3 签名和 4/16KB 对齐检查。

## v7.5 真机重点验收

### 存档目录与退出

- 点“存档槽”后应直接看到滚动槽位目录，而不是只有一段说明；每一行都应有“打开/继续”和“新游戏”。
- 点顶部“新建存档并开始新游戏”，输入新名称后应先保存当前游戏，再以新角色流程进入该槽。
- 打开另一个槽应先保存当前角色并重启载入；在已有槽点“新游戏”必须再次确认覆盖。
- 用原版正常保存退出后，只应看到“游戏已保存并正常退出”，不应出现返回码或诊断路径；轻触该画面应直接回到存档目录。
- 异常返回或启动失败时才应显示诊断日志路径。

### 保留操作区与跨区布局

- 确认游戏终端的最右侧文字完整停在操作区边界前，不被操作区背景或默认按键覆盖。
- 进入布局模式，把方向键从操作区拖到游戏区，再把普通快捷键拖回操作区；完成后两边按键均可正常触发。
- 拖操作区空白主体跨过屏幕中线，应在左/右侧切换停靠；拖黄色内边界应改变保留宽度，终端随之重新排布。
- 在操作区设置里修改颜色、形状、透明度和背景显示，重启后应完整保留。
- 退出布局模式后，顶部临时工具条、HUD 数量和拖拽说明应全部消失。
- 编辑两个按钮并输入相同 X，确认左边缘纵向对齐；再输入相同 Y，确认上边缘横向对齐。输入超出当前尺寸允许范围的数值时应阻止保存并显示范围。

### 输入与大键盘

- 短按 `8` 只走一格；按住超过 300ms 后持续向上，松手停止。
- 九宫格和自由按键按下时应变亮、缩小并产生系统允许的触感反馈。
- 把若干按键放到游戏区、另一些放到操作区，再打开大键盘：游戏区按键应不可见且不可点击，操作区按键仍可点击；关闭键盘后游戏区按键应原位恢复。
- 打开大键盘后点 `r` 应立即发送，不需要回车。
- 验证九方向、Ctrl 下一键/锁定、Shift 下一键/锁定，以及键盘大小、宽度、透明度和顶/底位置。

### 升级与恢复入口

- 从 v7.4 直接覆盖安装，确认存档、自由按键和操作区宽度保留。
- 删除全部 HUD 键后按 Android 返回键，应仍能打开游戏菜单并重新新增按键。
- 分别切换两个存档槽，确认先保存、短暂重启后载入目标槽。
